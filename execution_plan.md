# Parameter Golf 快速冲榜 Execution Plan

## Context
当前排行榜最高分 1.206 BPB，基线 1.2244 BPB。目标：在不使用外部模型的前提下，快速超越 1.206。资源：8×H100，每次实验 ≤10 分钟。

## 策略总览

按**投入产出比**排序，分三个阶段推进。每阶段以上一阶段最优配置为基础叠加。

---

## Phase 1: 低风险高回报（预计 1-2 天）

### 1.1 深度递归 + 加宽模型 ⭐⭐⭐
- **思路：** 将 9 层独立参数改为 3 组权重 × 循环 3 次（等效 9 层），省下约 2/3 的 Transformer 块参数，用来加宽 `model_dim`（512 → 640~768）
- **为什么优先：** 在 16MB 限制下这是最直接的参数效率杠杆，学术上已有充分验证（Universal Transformer, ALBERT）
- **对比实验：** 固定总参数预算，对比 {3组×3次, 2组×4次+1} 与 {加宽到 640, 704, 768} 的组合

### 1.2 序列长度 2048
- **思路：** `train_seq_len` 从 1024 改到 2048（排行榜第一已验证有效）
- **注意：** 显存增加，可能需要调整 batch size 或 grad_accum_steps
- **对比实验：** 1024 vs 2048，观察 BPB 和训练速度的权衡

### 1.3 学习率调度快速调优
- **思路：** 在最优模型结构上，快速扫描 warmdown_iters、matrix_lr、tied_embed_lr
- **范围：** matrix_lr ∈ {0.02, 0.04, 0.06}，warmdown_iters ∈ {800, 1200, 1600}

---

## Phase 2: 中等风险中等回报（预计 2-3 天）

### 2.1 分词器优化
- **思路：** 在 FineWeb 子集上重新训练 BPE 分词器，测试 vocab_size ∈ {1024, 2048, 4096}
- **关键：** BPB 指标的分母是字节数，词表大小直接影响每 token 的信息密度
- **权衡：** 词表越大嵌入矩阵越大，在 16MB 预算下需要仔细权衡

### 2.2 MLP 结构实验
- **思路：** 对比 ReLU² vs SwiGLU vs GeGLU，调整 expansion ratio
- **注意：** SwiGLU 参数多 50%（gate + up），需要在深度递归省出的预算内平衡

### 2.3 注意力配置调优
- **思路：** 调整 GQA 配比（8Q/4KV → 8Q/2KV 或 8Q/8KV），head_dim 调整
- **目标：** 在固定参数预算下找到最优注意力配置

---

## Phase 3: 探索性实验（预计 2-3 天）

### 3.1 量化感知训练（QAT）
- **思路：** 训练中模拟 int8 量化误差，减少训练-推理 gap
- **预期：** 量化后 BPB 下降更少，可能带来 0.001-0.005 的提升

### 3.2 RoPE base 调优
- **思路：** rope_base ∈ {5000, 10000, 50000, 100000}，特别是在 seq_len=2048 时
- **成本低：** 只改一个超参

### 3.3 Weight Decay 实验
- **思路：** 给 Adam 管理的参数加 weight decay ∈ {0.01, 0.05, 0.1}
- **目标：** 测试是否改善泛化

---

## 实验原则
1. **单变量控制：** 每次只改一个因素，用 baseline BPB 对比
2. **快速迭代：** 每次实验 ≤10 分钟，先看趋势再精调
3. **累积最优：** 每阶段的最优配置作为下一阶段的 baseline
4. **记录一切：** 每次实验记录超参、BPB、训练 loss 曲线
